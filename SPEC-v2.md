## Goal

Make librecrawl-mcp produce a Screaming-Frog / Ahrefs / Sitebulb-grade audit — every check, every edge case, paragraph-level content quality — delivered as a single **branded PDF**, via a **chunked progressive crawler** that never crashes and never overloads the target.

## Non-goals
- Replacing the LibreCrawl upstream crawler (we stay a wrapper)
- Building a SaaS dashboard (PDF + CSV files are the deliverable)
- Backlinks / keyword-volume parity with Ahrefs (no data moat to match)

## Success criteria
1. `librecrawl_full_audit_strict(url)` produces `.pdf` + `.md` + `.per-page.csv` + `.sitemap-recon.csv` in REPORTS_DIR.
2. PDF footer: *"Report Generated by LibreCrawl MCP — By Aditya Sharma · github.com/adityaarsharma/librecrawl-mcp"* (NO POSIMYTH branding).
3. Audit on 5,000-page site completes without crashing MCP client.
4. ≥95% Screaming Frog free-tier coverage.
5. Smoke-tested on a real POSIMYTH site (TPAE / uichemy / SHE).

## Architecture (from research agent #12)

```
┌──────────────┐   start_chunked_audit         ┌────────────────────────┐
│  MCP Client  │ ─────────────────────────────▶│  server.py (FastMCP)   │
│  (Claude)    │ ◀─── session_id + chunk_1 ────│  :5081 HTTP            │
└──────────────┘                                └──────────┬─────────────┘
       │                                                   │ enqueue job
       │  audit_status(id)  (every 10-30s, <60s each)      ▼
       │ ◀──────────────────────────────────────  ┌──────────────────────┐
       │                                          │  Job Runner (asyncio │
       │                                          │  task, resumable)    │
       │                                          │  - adaptive sizer    │
       │                                          └──────┬───────────────┘
       │                                                 │ POST /api/start_crawl (per chunk)
       │                                                 ▼
       │                                          ┌──────────────────────┐
       │                                          │ LibreCrawl Flask     │
       │                                          │ :5080 (bounded)      │
       │                                          └──────┬───────────────┘
       │                                                 ▼
       │                                          ┌──────────────────────┐
       │                                          │ SQLite state.db WAL  │
       │                                          │  sessions / chunks   │
       │                                          │  frontier / metrics  │
       │                                          │  artifacts           │
       │                                          └──────────────────────┘
```

### State machine

```
queued ──▶ crawling ──▶ chunk_complete ──┬──▶ next_chunk ──▶ crawling
                  │                       └──▶ done (auto-finalize)
                  ├──▶ throttled ──(backoff)──▶ crawling
                  ├──▶ paused ──▶ crawling
                  └──▶ failed (terminal)
```

### MCP tool surface (final)

**New (chunked engine):**
- `librecrawl_start_chunked_audit(url, total_max_pages=10000, chunk_target_pages=50, politeness="auto", confirm_unbounded=False)` → `{session_id, first_chunk_summary}`. Returns < 2s.
- `librecrawl_audit_status(session_id)` → progress + adaptive metrics. Sole polling tool.
- `librecrawl_audit_artifacts(session_id)` → `{md, pdf, csvs[]}` paths once `status=done`.
- `librecrawl_audit_pause(session_id)` / `librecrawl_audit_resume(session_id)` / `librecrawl_audit_cancel(session_id)` — operator control.
- `librecrawl_audit_force_advance(session_id)` — escape hatch for stuck sessions.

**Backwards compat (unchanged signatures):** `librecrawl_audit`, `librecrawl_full_audit_strict` — become thin facades over chunked engine with 110s hard timeout, returning partial-result envelope when crawl exceeds.

### Adaptive sizing (AIMD)

```
TARGET_P95_MS = 1500;  TARGET_ERR_RATE = 0.02
On Retry-After:        size /= 2, delay = max(delay, retry_after_ms)
On err > 10%:          size /= 2, delay = delay*2 + 500
On err > 2%:           size *= 0.75, delay += 250
On p95 > 1.5x target:  size *= 0.7,  delay *= 1.5 + 100   (MD)
On p95 < 0.6x + clean: size += 10,   delay -= 100          (AI)
Clamp: size [10,200], delay [robots_crawl_delay, 5000ms]
EWMA inputs alpha=0.4
```

### Failure modes
- 5xx storm → throttled, halve size, resume in 30s
- Retry-After ≥ 5min → paused
- Cloudflare 429 → treat as Retry-After=120s; 3 consecutive → paused
- PM2 restart mid-chunk → on boot, reset in-flight frontier to pending, resume
- LibreCrawl :5080 down → exp backoff 5/15/45s, then paused
- robots disallow root → failed immediately with reason

### New Python files
- `runner.py` — asyncio job runner, chunk loop, adaptive controller
- `state.py` — SQLite schema (sessions/frontier/chunks/artifacts), WAL config
- `libreclient.py` — typed wrapper around LibreCrawl with per-request metrics
- `artifacts.py` — MD/PDF/CSV writers consuming cumulative SQLite state
- `pdf_template.py` — WeasyPrint HTML/CSS template with Aditya branding
- `content_audit.py` — readability, AI tells, missing punctuation, boilerplate
- `extended_checks.py` — security headers, mixed content, soft-404, hreflang, sitemap cross-checks

## Content audit (Aditya: "every line, paragraph, missing period")

All computed server-side from already-fetched HTML — zero token cost.

- **Flesch reading ease** per page (flag < 50)
- **Avg sentence length** (flag > 25 words avg)
- **Passive voice ratio** (flag > 25%)
- **Missing terminal punctuation** (sentence not ending `.!?`)
- **Double spaces** in body text
- **Smart-quote mismatches** (`"` paired with `'` or vice versa)
- **Boilerplate ratio** via shingle overlap across pages (flag > 70%)
- **Thin-content category** (low unique-text ratio after boilerplate strip)
- **AI-tell tokens** (delve / unlock / seamlessly / leverage / unleash / em-dash density)
- **Lorem ipsum detection** (placeholder text shipped to prod)
